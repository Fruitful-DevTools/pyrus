Project Overview

Pyrus is a Python library designed to simplify common development tasks by providing a common interface for performing operations across different mediums. It includes features such as file management, web requests, data manipulation, and audio processing, among others. The library is designed to be simple and easy to use, with a focus on providing concise and readable code.

One of the key goals of Pyrus is to reduce the amount of boilerplate code that developers need to write when performing common tasks. For example, the file management module provides a simple interface for reading and writing files, handling file compression, and manipulating file paths. Similarly, the web module provides a simple interface for making HTTP requests and handling responses, including handling common errors and retries. Pyrus also includes a caching module, which provides a simple way to cache expensive computations or API requests for faster and more efficient processing.

Overall, Pyrus aims to make it easier for developers to write efficient, maintainable code by providing a set of common tools and abstractions for common tasks. By reducing the amount of boilerplate code that developers need to write, Pyrus allows developers to focus on the logic of their code, rather than the details of how to perform common tasks.

pyrus.file_manager

pyrus.file_manager is a utility module that enables you to read and write different types of files. It provides you with the flexibility to read and write data tables, images, videos, audios, and text files in the formats supported by the module.

Usage

To use the module, you can import the FileManager class as shown below:

from file_manager import FileManager

You can then create an instance of the class by specifying the path of the file that you want to read or write. For example, to read data from a CSV file, you can create an instance of the FileManager class and call the read() method, as shown below:

fm = FileManager('data.csv') fm.write(data_to_write_to_file, add_date_to_filename=True)

The write() method takes two arguments:

  • data_to_write_to_file: the data that you want to write to the file.
  • add_date_to_filename: a Boolean value that determines whether to add the current date and time to the file name.

Supported File Types

The file_manager module supports the following file types and extensions:

  • Data Table: CSV (.csv), JSON (.json), Excel (.xlsx, .xls)
  • Image: PNG (.png), JPG (.jpg), Bitmap (.bmp), GIF (.gif), TIFF (.tiff), WebP (.webp), PPM (.ppm)
  • Video: mp4 (.mp4), avi (.avi), wmv (.wmv), flv (.flv), mkv (.mkv), ogv (.ogv), mpeg (.mpeg), mov (.mov)
  • Audio: mp3 (.mp3), wav (.wav), aac (.aac), flac (.flac), aiff (.aiff), ogg (.ogg)
  • Text: Microsoft Word (.doc, .docx)
